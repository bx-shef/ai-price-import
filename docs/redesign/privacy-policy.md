Настоящая Политика описывает, какие данные обрабатывает приложение «procure-ai» (далее — «Приложение»)
при работе в облачном сервисе Bitrix24, с какой целью, куда они передаются, сколько хранятся и какие
права есть у пользователя. Устанавливая и используя Приложение, администратор портала Bitrix24
подтверждает, что ознакомлен с настоящей Политикой.

---

## 1. Какие данные обрабатываются

Приложение обрабатывает данные **только по инициативе пользователя** (загрузка документа) и служебные
данные, необходимые для интеграции с порталом Bitrix24:

| Категория | Что именно | Источник |
|---|---|---|
| **Документ с табличной частью** | Загруженный пользователем файл (накладная / счёт / КП / прайс): исходный бинарный файл и извлечённый из него текст/структура | Загрузка пользователем в интерфейсе Приложения |
| **Данные контрагентов из документа** | Наименования поставщиков, реквизиты (УНП/ИНН/БИН), номенклатура, цены, суммы, НДС — то, что содержится в документе | Содержимое загруженного документа |
| **Идентификатор портала** | `member_id` портала Bitrix24 и его домен | Событие установки / фрейм-токен Bitrix24 |
| **OAuth-токены портала** | `access_token` / `refresh_token` / `application_token` для вызовов REST API портала | OAuth-авторизация Bitrix24 при установке |
| **Настройки Приложения** | Карта сопоставления (каталог, поле артикула, целевая сущность/направление, чаты уведомлений) | Ввод администратора портала |
| **Служебные счётчики** | Пожизненные агрегаты по порталу (сколько документов обработано/создано/не сопоставлено) — без содержимого документов | Формируются автоматически при обработке |

Приложение **не запрашивает** и не собирает специальные категории персональных данных сверх того, что
пользователь сам поместил в загружаемый документ или в настройки портала.

## 2. Цели обработки

- Автоматическое распознавание табличной части документа и создание соответствующей сущности в CRM
  портала (сделка/лид/смарт-процесс с товарными позициями и НДС).
- Поиск контрагента в CRM портала по реквизитам из документа.
- Сохранение исходного файла на Диске портала и создание дела/уведомления в чат.
- Поддержание авторизации Приложения в портале (OAuth) и обеспечение работоспособности интеграции.
- Ведение обезличенных эксплуатационных счётчиков и (при явном включении администратором) технической
  телеметрии для диагностики и улучшения качества сервиса.

## 3. Куда передаются данные (важно)

### 3.1. Текст документа — на внешний LLM-провайдер

Для распознавания табличной части **извлечённый из документа текст** (включая наименования и реквизиты
контрагентов, номенклатуру и цены) передаётся на инференс **внешнему провайдеру больших языковых
моделей (LLM)**. Исходный бинарный файл на LLM **не передаётся** — только извлечённый текст.

> **Провайдер публичной версии — BitrixGPT** (AI Router сервиса Битрикс24 Вайбкод): извлечённый текст
> передаётся на инференс в инфраструктуру Битрикса, юрисдикцию данных на этом инференсе несёт Битрикс.
> Оператор Приложения (self-hosted) может выбрать иного провайдера настройкой окружения (напр. DeepSeek —
> юрисдикция КНР — только с информированным согласием клиента по 152-ФЗ/коммерческой тайне). Актуальный
> провайдер указывается в карточке Маркета. ‹Владелец: сверить формулировку юрисдикции BitrixGPT с
> условиями Битрикс24 перед публикацией.›

### 3.2. Данные, остающиеся внутри периметра

| Данные | Куда | Примечание |
|---|---|---|
| Исходный файл документа | **Диск портала Bitrix24 клиента** | Наружу не передаётся; хранится внутри портала клиента |
| OAuth-токены портала | База данных backend издателя (Postgres) | `refresh_token` шифруется (AES-256-GCM); `application_token` — write-once |
| Служебные счётчики | База данных backend издателя | Скоуп per-portal; без содержимого документов |
| Настройки Приложения | `app.option` портала Bitrix24 | Хранятся средствами портала |

### 3.3. Отзывы сотрудников (по инициативе пользователя)

Если пользователь отправляет отзыв (👍/👎/💡) о результате обработки, текст отзыва и технический контекст
(что распозналось, какие возникли нестыковки) передаются в **закрытый (приватный) репозиторий издателя**.
**Исходный файл документа прикладывается к отзыву только с явного согласия** отправителя (по умолчанию —
без файла). Отзывы используются исключительно для устранения дефектов и улучшения качества сервиса.

### 3.4. Телеметрия (по умолчанию выключена)

Техническая телеметрия (OpenTelemetry) **по умолчанию отключена**. При включении администратором она
передаёт на коллектор издателя только «форму» операций (длительность, статус, обезличенные счётчики) и
**не содержит** коммерческих данных: идентификатор портала передаётся необратимым хешем, содержимое
документа/поставщик/артикул/цена в телеметрию не попадают (техническая защита на уровне кода).

Приложение **не продаёт** данные третьим лицам и **не использует** их для рекламного профилирования.

## 4. Сроки хранения и удаление

- **Статус задания импорта** хранится во временном хранилище (Redis) и удаляется автоматически по
  истечении срока жизни (по умолчанию — 48 часов).
- **Извлечённый текст и структура документа** удаляются штатно в течение нескольких минут после обработки;
  дополнительно действует резервная очистка по расписанию (TTL-свип, по умолчанию — старше 24 часов).
- **Исходный файл** хранится на Диске портала клиента и управляется клиентом (издатель его не выносит
  наружу за исключением п. 3.3 с согласия).
- **OAuth-токены** хранятся в зашифрованном виде до удаления Приложения.
- **Удаление Приложения с портала** (`ONAPPUNINSTALL`) влечёт удаление данных этого портала на стороне
  издателя, хранящихся в базе данных: OAuth-токены, служебные счётчики, извлечённые текст/структура
  документов. Статусы заданий импорта хранятся во временном хранилище (Redis) и истекают автоматически
  по TTL (по умолчанию — в течение 48 часов).

Полный текст документов и секреты **не записываются** в общие журналы.

## 5. Права пользователя

- Получить сведения о том, какие данные обрабатываются, и цели обработки (по обращению к контакту выше).
- Прекратить обработку в любой момент — **удалив Приложение** с портала (влечёт удаление данных портала,
  п. 4).
- Запросить удаление конкретных данных или отзыва до истечения срока их хранения (по обращению).
- Отказаться от передачи текста на внешний LLM — не загружая документы (обработка инициируется только
  загрузкой).

## 6. Безопасность

- `refresh_token` портала шифруется в покое (AES-256-GCM); `application_token` пишется однократно.
- Все вызовы к порталу идут по защищённому транспорту с проверкой домена (защита от подмены).
- Доступ к настройкам и служебным операциям ограничен ролью администратора портала.
- Приложение минимизирует хранение: клиентские данные хранятся в минимальном объёме и удаляются на
  терминальных путях обработки и при удалении Приложения (передача текста на внешний LLM описана в п. 3.1).

## 7. Self-hosted (разворачивание на сервере клиента)

При установке Приложения на инфраструктуру клиента (self-hosted) приложение, база данных, очередь и
временные файлы остаются у клиента. **Однако текст документа всё равно передаётся на внешний LLM-инференс**
(п. 3.1) — self-hosted сам по себе не делает обработку полностью локальной. Полностью замкнутый контур
(локальная/он-прем LLM) — предмет отдельного соглашения. Ответственность за аптайм/бэкапы/патчи/защиту
ключей на инфраструктуре клиента определяется отдельным договором.
