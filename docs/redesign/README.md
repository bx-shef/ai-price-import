# Редизайн procure-ai — документация

> Last reviewed: 2026-07-21

Рабочий набор документов по переделке проекта AI-импорта прайсов с текущей 4-сервисной
раскладки на единый Nuxt 4 + Nitro монолит (по образцу `client-bank-alfa-by`), с сохранением
AI-слоя (агент-экстрактор). **Кода для ядра/коробки Б24 нет** — вся интеграция через
стандартный REST по OAuth-токену портала (решение Q7). Агент **tool-less** (без
MCP-HTTP-сервера): распознаёт документ в JSON, а детерминированные тела инструментов
(поиск/запись) исполняет `crm-sync` в процессе — так безопаснее против prompt-injection
(решение из ревью, см. [`02-target-architecture.md`](02-target-architecture.md) §6).

## Референсы

- **Эталон архитектуры/лендинга:** `client-bank-alfa-by` (Nuxt 4 + Nitro, проверен на проде).
- **Методология AI-решений под Б24:** `ai-agent` (`docs/09-tz/00_intake-ai/`, `docs/15-integrations/`) —
  принцип «изолированный MCP», шаблон ТЗ, паттерны интеграции.
- **Старый проект (as-is):** этот же репозиторий, слои `backend/` `mcp/` `mcp-overlay/` `ui/` `b24-controller/`.

## Статус

Каркас монолита построен: `server/` (agent, queue, api, db, plugins) + `app/` (лендинг,
in-portal и служебные страницы) — с покрытием тестами (`pnpm check` зелёный). Готовы 🧪 приём
событий Б24, хранилище токенов, доменное ядро, пайплайн extract→agent→crm-sync (на заглушках
транспортов), операторская авторизация, лендинг и публичное демо. В работе — сквозная интеграция
на живом портале (реальная запись в CRM, диск, дело, чат), опрос статуса и деплой (этапы 3/4/8/9
карты). Актуальный срез — [`01-project-map.md`](01-project-map.md).
