# Демо на лендинге — карта продукта и журнал работ

> Last reviewed: 2026-07-09

Публичное демо на лендинге: пользователь прикрепляет файл (КП / счёт / ТТН) → получает ответ
«что за подрядчик, какие товары и т.п.». Плюс готовые примеры (рус/бел/каз) и возможность
загрузить свой файл. **Дизайн/тексты/UX правим потом — сейчас главное, чтобы всё отрабатывало.**

Этот документ — источник правды по демо. **Перечитывать перед каждым новым куском работы** и
держать журнал внизу синхронно.

## Требования (из задачи владельца)

1. Прикрепить файл → ответ: подрядчик (поставщик), товары, тип документа, суммы.
2. Дать 2–3 примера (или сгенерировать самому): КП, счёт, ТТН на рус/бел/каз.
3. Пользователь может загрузить свой файл — **предупредить о публичности** и **ограничить
   3 файла за 10 минут**.
4. Баннер-текст: «если нужно доработать — сделаем на ваших серверах».
5. Всё должно реально отрабатывать (UX/дизайн — потом).

## Ограничения окружения (проверено вживую)

- **DeepSeek/LLM-ключа нет** в окружении → демо **не** зависит от платного LLM.
- В dev-контейнере нет `pdftotext`/`tesseract` (есть `libreoffice`). Прод-образ их несёт, но
  публичное демо не должно упираться в тяжёлый OCR/агента.
- Bitrix24 в демо **не** участвует (ни токенов, ни записи) — только разбор и показ.

## Архитектурное решение

Демо = **детерминированный экстрактор** по полуструктурированным текстовым документам
(заголовок + таблица товаров), мультиязычный (рус/бел/каз). Без LLM, без Б24, без записи.

- **Чистое ядро** `app/utils/demoExtract.ts` (покрыто тестами): определить тип документа
  (КП/счёт/ТТН по ключевым словам трёх языков), поставщика (метка + налоговый ID УНП/ИНН/БИН),
  таблицу товаров (синонимы колонок Наименование/Кол-во/Цена/Сумма), итоги.
- **Backend** `POST /api/demo/extract`: rate-limit 3 файла / 10 мин на IP, лимит размера,
  флаг публичности; декод + вызов чистого ядра; ответ — JSON `DemoResult`.
- **Готовые примеры** — статические файлы в `public/demo/*` (9 штук: КП/счёт/ТТН × ru/be/kk),
  кнопки «попробовать на примере».
- **UI** — секция на лендинге (`app/pages/index.vue` / компонент `DemoTryout.vue`): дропзона +
  кнопки примеров + карточка результата + баннер про доработку.
- **Заложить переключатель** на реальный агент: когда появится LLM-ключ и прод-бинарями,
  endpoint сможет вызвать существующий `runAgent` вместо детерминированного ядра.

## Формат демо-файлов

Реалистичные, но детерминированно-парсимые текстовые документы (`.txt`, кодировка UTF-8):
заголовок с типом и номером, блок «Поставщик … УНП/ИНН/БИН …», таблица товаров с шапкой
(Наименование | Кол-во | Ед. | Цена | Сумма) и итоговой строкой. По одному набору на язык.

## Безопасность демо

- Публичность: предупреждение в UI + в ответе; файлы не сохраняем дольше обработки.
- Rate-limit 3/10мин на IP (in-memory окно; за прокси — по `X-Forwarded-For` первый хоп).
- Лимит размера файла; декод только текстовых форматов; никакого исполнения.
- Никаких секретов/Б24 в демо-пути.
