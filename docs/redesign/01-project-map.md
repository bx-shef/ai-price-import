## 1. Цель

Переделать AI-импорт прайсов (`procure-ai`) с текущей 4-сервисной раскладки на
**единый Nuxt 4 + Nitro монолит** по образцу `client-bank-alfa-by`, сохранив ценность
(агент + промпт + MCP-инструменты + PHP-контроллеры коробки) и добавив то, чего нет:
дисциплину чистого ядра с тестами, очередь под нагрузку, публичный лендинг и маркетинг.

**Критерий готовности MVP редизайна:** менеджер загружает накладную на новом стеке →
агент через изолированный MCP создаёт сделку «Закупки» на портале заказчика; лендинг
опубликован; деплой одним пайплайном (GHCR + Watchtower).

---

## 2. Целевая архитектура (одной фразой)

Nuxt-монолит: **лендинг + in-portal UI + Nitro-backend** в одном приложении; backend
оркеструет **агента Claude Code**, который ходит в Bitrix24 **только через изолированный MCP**;
MCP зовёт PHP-контроллеры коробки. Тяжёлые шаги (извлечение текста, прогон агента, запись в CRM) —
через **очередь BullMQ + Redis**. Детали — [`02-target-architecture.md`](02-target-architecture.md).

## 5. Что переносим из старого проекта

| Актив | Куда в новой раскладке | Примечание |
|---|---|---|
| Промпт `prompts/main.md` | `prompts/` или `server/agent/` | Ценность как есть; ревизия правил (НДС, единицы) |
| 5 MCP-инструментов `b24_pst_crm_*` | отдельный MCP-пакет, первоклассный код + тесты | Уйти от вендоренного git-subtree + overlay |
| Ядро извлечения текста | `server/` + утилиты | pdftotext/tesseract/office; в очереди |
| PHP-контроллеры `procure*.php` | `b24-controller/` (переносим 1-в-1) | Контракт Node↔PHP; деплой на теге |
| Homoglyph-логика | `app/utils/*` (чистая, тесты) | Уже почти чистая функция |
| Eval-харнесс точности | `tests/eval` или отдельный прогон | Сохранить сигнал «направление НДС» |
| Метрики/feedback | `server/` + Postgres/Redis | Пересобрать на очереди/БД |

**Не переносим:** мёртвый чат-бот (#241), выключенный OAuth-стор `better-sqlite3`,
вендоренный `mcp/` как git-subtree, i18n на 19 локалей (если не нужен — упростить до `ru`).
