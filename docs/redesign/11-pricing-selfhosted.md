# Тарифы, калькулятор и self-hosted оффер (procure-ai)

> Last reviewed: 2026-07-22

Спецификация для будущего **калькулятора расчётов** и чек-лист по **self-hosted** предложению
(«развернём на вашем сервере»). Появилось из #96 (ревью позиционирования, см.
[04-marketing §2](04-marketing-landing.md) и [05-data-policy §3a](05-data-policy.md#3a-self-hosted-контур-разворачивание-на-сервере-клиента)). **Это черновик
требований, а НЕ утверждённые цены/условия** — финальные цифры и юр-модель за владельцем/юристом.

## 1. Две модели поставки

| | Облако (подписка Маркета) | Self-hosted (на сервере клиента) |
|---|---|---|
| Инфраструктура | общий минимальный сервер издателя | выделенный сервер клиента |
| Производительность | **ограниченный параллелизм**, в пик — очередь (см. [09-deploy §Ресурсы воркера](09-deploy.md#ресурсы-воркера-минимум--стабильность-осознанный-выбор)) | параллельно, без соседей-арендаторов |
| Данные | документы/токены у издателя; **текст → LLM наружу** | всё у клиента, но **текст всё равно → LLM наружу** (см. §3) |
| Цена | дешевле, подписка | дороже, разовое разворачивание + сопровождение |
| Кому | большинство | большие объёмы / требования по скорости |

**Честный месседж (вшить в тексты):** облако — «надёжно, но не мгновенно»; нужна скорость →
self-hosted. Не обещать «за секунды».

## 2. Калькулятор — требования

Компонента пока **нет** — это ТЗ на неё (в бэклоге: docs/ROADMAP §4.0):

- **Развилка вверху:** «Облако» ↔ «На вашем сервере».
- **Единица расчёта — определить** (выбрать одну, зафиксировать с владельцем):
  за документ / за пользователя портала / фикс в месяц / гибрид (абонплата + пакет документов).
- **Облако:** тариф(ы) подписки + лимиты (док/мес, размер файла, скорость best-effort).
- **Self-hosted:** для **РФ** — по референс-модели (внутренний ориентир, НЕ клиентская формулировка,
  см. сноску в 04-marketing); для **РБ** — плейсхолдер «по запросу», пока юр-условия не уточнены (§4).
- Выводить **оговорку про данные** (§3) прямо в результате калькулятора для self-hosted.

## 3. Данные при self-hosted (критично — не ввести в заблуждение)

Self-hosted **сам по себе НЕ делает контур замкнутым**. Что остаётся у клиента и что уходит —
см. [05-data-policy §3a](05-data-policy.md#3a-self-hosted-контур-разворачивание-на-сервере-клиента). Кратко: приложение/БД/очередь/файлы — у клиента, **но
текст документа уходит на LLM-инференс внешнему провайдеру** (по умолчанию BitrixGPT — юрисдикция Битрикса; DeepSeek/КНР — опция). Полностью
локальный контур = **отдельный разговор про он-прем LLM** (поднять/оплатить модель отдельно; слой
провайдер-агностичен, `ANTHROPIC_*`).

## 4. Открытые вопросы (перед выносом в реальный оффер)

- **Юр-модель РФ/РБ:** для РФ — по референс-модели; для РБ — юрисдикция/хостинг/договор уточнить (юрист).
- **Ответственность:** аптайм/бэкапы/патчи/шифрование ключей на инфраструктуре клиента — зона клиента;
  как едут обновления образа (Watchtower у клиента? ручной релиз?), SLA.
- **Единица и уровни цен** калькулятора (§2) — решение владельца.
- **Данные:** оффер полностью-локального (он-прем LLM) контура — отдельный SKU?

Владелец РФ-модели — издатель (ИП Шевчик И.С.); РБ — по запросу. Связано: GH #96, #71 (инфра
разворачивания).
